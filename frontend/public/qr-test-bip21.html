BIP21 Format Experiment

Testing: Can we put a paymail or identity key in the BIP21 bitcoin: address position and still get recipient + amount?

Test 1: BIP21 with Paymail + Amount

bitcoin:user@handcash.io?amount=0.005&label=Paymail BIP21 Test
BIP21 URI with paymail in address position
Question: Does recipient populate as "user@handcash.io" AND amount as 0.005?
What we want to see: recipient = user@handcash.io, amount = 0.005 BSV, paymail resolution triggers

Test 2: BIP21 with Paymail (no amount)

bitcoin:user@handcash.io
BIP21 URI with paymail only, no amount
Question: Does recipient populate as "user@handcash.io" with blank amount?

Test 3: BIP21 with Identity Key + Amount

bitcoin:02c0b24547...d3c1?amount=0.01&label=Identity Key BIP21
BIP21 URI with compressed public key (66 hex chars) in address position
Question: Does recipient populate as the identity key AND amount as 0.01?
What we want to see: recipient = 02c0b245...d3c1 (full key), amount = 0.01 BSV, PeerPay route triggers

Test 4: BIP21 with $handle + Amount

bitcoin:$testhandle?amount=0.002&label=Handle BIP21
BIP21 URI with HandCash $handle in address position
Question: Does recipient populate as "$testhandle" AND amount as 0.002?
