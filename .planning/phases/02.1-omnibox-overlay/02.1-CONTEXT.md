# Phase 2.1: Omnibox Overlay - Context

**Created:** 2026-01-25
**Status:** Ready for planning

<vision>
## How This Should Work

Chrome-style behavior: clicking the address bar opens a unified overlay containing both the input and dropdown suggestions in a single rounded pill-shaped container. The overlay appears instantly on top of everything else, positioned exactly where the address bar is.

When you click the address bar placeholder in MainBrowserView, it should feel like the address bar "expands" into a focused state with the dropdown already visible (showing all suggestions), similar to Chrome's omnibox behavior.

The overlay disappears when you navigate or press Escape, returning to the simple placeholder view.
</vision>

<essential>
## What Must Be Nailed

- **Instant overlay appearance** - No delay, feels like the address bar is transforming
- **Correct positioning** - Overlay positioned exactly at the top where the address bar placeholder is
- **Unified pill container** - Single rounded box wrapping both input and dropdown (not two separate elements)
- **Chrome-like behavior** - Click shows all suggestions immediately, typing filters them
- **IPC communication** - Overlay sends navigation commands back to main browser
</essential>

<boundaries>
## What's Out of Scope

- **Real database integration** - Still using mock suggestions (Phase 3)
- **Google search** - Phase 4 feature
- **Advanced keyboard navigation** - Arrow keys can be basic for now, Phase 5 will polish
- **Tab sync** - Deferred to Phase 3
</boundaries>

<specifics>
## Implementation Approach

Follow the wallet panel pattern but simplified:

1. **C++ Side:**
   - Create OmniboxOverlay window (WS_EX_LAYERED | WS_EX_TOPMOST like wallet)
   - Position at top of main window (y=0, height=~300px to fit dropdown)
   - Show/hide via IPC messages

2. **React Side:**
   - OmniboxOverlayRoot.tsx component
   - Reuse existing Omnibox component inside unified pill container
   - Auto-focus input when overlay opens
   - Send navigation via IPC, close overlay after

3. **MainBrowserView:**
   - Replace Omnibox with simple placeholder
   - Placeholder click → send IPC to show overlay
   - Display current URL (read-only or minimal interaction)

This solves the z-index issue completely and gives better UX.
</specifics>
