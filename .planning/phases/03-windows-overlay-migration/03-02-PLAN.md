<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Windows HodosBrowser with updated overlay system - wallet panel, advanced features, settings, backup, and BRC-100 auth overlays now use the same React UI as macOS
  </what-built>
  <how-to-verify>
Prerequisites:
- Rust wallet running: `cd rust-wallet && cargo run --release` (localhost:3301)
- Frontend dev server running: `cd frontend && npm run dev` (localhost:5137)

1. **Start Windows browser**:
   - `cd cef-native/build/bin/Release`
   - `./HodosBrowserShell.exe`

2. **Test Wallet Panel** (from Phase 1):
   - Open wallet panel (button or shortcut)
   - Verify: Balance displays correctly
   - Verify: Send transaction form appears with prompt
   - Verify: Receive address retrieval works with prompt
   - Verify: Overlay window has transparency and proper layering

3. **Test Advanced Features** (from Phase 1):
   - Click "Advanced Features" or equivalent
   - Verify: Opens in new tab (not modal)
   - Verify: Tabbed interface shows Transactions, Addresses, UTXOs, Certificates
   - Verify: Each tab loads data when selected
   - Verify: Per-tab refresh buttons work

4. **Test Settings Overlay**:
   - Open settings
   - Verify: Settings overlay appears with proper styling
   - Verify: Settings options are functional

5. **Test DevTools** (from Phase 2):
   - Press F12 or Ctrl+Shift+I
   - Verify: DevTools window opens in detached mode
   - Verify: DevTools shows correct browser context
   - Right-click on page, select DevTools from context menu
   - Verify: Works consistently

6. **Test Backup Overlay** (if applicable):
   - Trigger backup flow
   - Verify: Backup modal appears correctly

7. **Check Windows-specific behavior**:
   - Verify: Window chrome looks native to Windows
   - Verify: Overlay shadows render correctly (Windows DWM)
   - Verify: No visual glitches or rendering artifacts
   - Verify: Keyboard shortcuts feel Windows-appropriate (Ctrl+Shift+I not Cmd+Option+I)

8. **Confirm no regressions**:
   - Main browser navigation still works
   - No crashes when opening/closing overlays
   - No memory leaks (open/close overlays multiple times)

Expected: All overlays work identically to macOS version, Windows-specific UI conventions are respected
  </how-to-verify>
