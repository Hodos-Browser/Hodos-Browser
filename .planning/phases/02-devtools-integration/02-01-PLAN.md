**Established patterns:**
- Role-based logic in SimpleHandler (`role_` member: "wallet", "settings", etc.)

- Platform-specific code with `#ifdef _WIN32` / `#elif defined(__APPLE__)`

**Constraining decisions:**
- Phase 1: macOS wallet UI complete, overlay system established
- Current DevTools implementation only for overlays via right-click "Inspect Element"
- DevTools window created with `nullptr` for CefWindowInfo (needs fixing per research)

**Research findings:**
- Use `SetAsPopup(NULL, "DevTools")` for detached windows (avoids blank window issues)
- F12 works cross-platform via `VK_F12` key code
- Cmd+Option+I for macOS: check `windows_key_code == 'I'` with `EVENTFLAG_COMMAND_DOWN` and `EVENTFLAG_ALT_DOWN`
- Ctrl+Shift+I for Windows: check `windows_key_code == 'I'` with `EVENTFLAG_CONTROL_DOWN` and `EVENTFLAG_SHIFT_DOWN`
- Call `HasDevTools()` before `ShowDevTools()` to avoid multiple windows
- Each browser instance (main + overlays) gets independent DevTools
</context>

  <name>Task 1: Implement keyboard shortcuts for DevTools</name>
  <files>cef-native/src/handlers/simple_handler.cpp, cef-native/include/handlers/simple_handler.h</files>
  <action>
    Modify `SimpleHandler::OnPreKeyEvent()` to handle DevTools keyboard shortcuts for ALL windows (not just overlays):

    1. Remove the early return for overlay windows - they should also support keyboard shortcuts
    2. Add F12 detection: `if (event.type == KEYEVENT_RAWKEYDOWN && event.windows_key_code == VK_F12)`
    3. Add macOS Cmd+Option+I: Check `windows_key_code == 'I'` with `EVENTFLAG_COMMAND_DOWN` and `EVENTFLAG_ALT_DOWN` modifiers (use `#ifdef __APPLE__`)
    4. Add Windows Ctrl+Shift+I: Check `windows_key_code == 'I'` with `EVENTFLAG_CONTROL_DOWN` and `EVENTFLAG_SHIFT_DOWN` modifiers (use `#ifdef _WIN32`)
    5. For each shortcut: Call helper function `ShowOrFocusDevTools(browser)` and return true to consume event
    6. Create private helper method `ShowOrFocusDevTools(CefRefPtr&lt;CefBrowser&gt; browser)` that:
       - Checks `if (!browser->GetHost()->HasDevTools())`
       - Creates `CefWindowInfo` and calls `SetAsPopup(NULL, "DevTools")` (cross-platform)
       - Calls `browser->GetHost()->ShowDevTools(windowInfo, browser->GetHost()->GetClient(), settings, CefPoint())`
       - Logs with `LOG_DEBUG_BROWSER("🔧 DevTools opened via keyboard shortcut")`

    **Why these changes:**
    - F12 is universal developer shortcut, works cross-platform
    - Cmd+Option+I matches Chrome on macOS (standard developer muscle memory)
    - Ctrl+Shift+I is Windows alternative to F12
    - SetAsPopup prevents blank window issues (research finding from CEF forums)
    - HasDevTools check prevents multiple DevTools windows for same browser
    - Removing overlay-only restriction allows main browser DevTools access

    **What to avoid:**
    - Don't return true for all keys (breaks text input) - only consume DevTools shortcuts
    - Don't use SetAsChild for DevTools window (causes blank windows on Windows)
    - Don't call ShowDevTools without checking HasDevTools (creates duplicate windows)
  </action>
