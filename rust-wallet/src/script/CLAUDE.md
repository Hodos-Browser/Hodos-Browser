# Script — Bitcoin Script Parsing and PushDrop Encoding (BRC-48)

> Bitcoin Script chunk parser and PushDrop (BRC-48) encode/decode for embedding arbitrary data in spendable transaction outputs.

## Overview

This module provides two core capabilities: (1) a Bitcoin Script parser that decomposes raw script bytes into typed chunks (opcodes and data pushes), and (2) a PushDrop encoder/decoder implementing BRC-48 — the pattern used to embed certificate data and other arbitrary fields into transaction outputs while keeping them spendable via `OP_CHECKSIG`. The PushDrop format is central to BRC-52 identity certificates, where certificate JSON is stored as the first field.

**Security note**: This module handles script construction and parsing only — it does not perform signing or hold private keys.

## Files

| File | Purpose |
|------|---------|
| `mod.rs` | Module root — re-exports core types from `pushdrop` and `parser` |
| `parser.rs` | Bitcoin Script chunk parser: `parse_script_chunks()`, `ScriptChunk`, `opcodes` constants, `ScriptParseError` |
| `pushdrop.rs` | BRC-48 PushDrop encode/decode: `encode()`, `decode()`, `create_minimally_encoded_chunk()`, `PushDropDecoded`, `LockPosition`, `PushDropError` |
| `pushdrop_tests.rs` | Cross-implementation tests against TypeScript SDK test vectors and Bitcoin standard opcodes |

## Key Exports

### Script Parser (`parser.rs`)

| Export | Description |
|--------|-------------|
| `ScriptChunk` | Single script element: `op: u8` (opcode) + `data: Option<Vec<u8>>` (pushed bytes, if any) |
| `parse_script_chunks(script: &[u8])` | Parse raw Bitcoin script bytes into `Vec<ScriptChunk>`. Handles direct pushes (1-75 bytes), `OP_PUSHDATA1/2/4`, and bare opcodes (`OP_0`, `OP_1`-`OP_16`, `OP_1NEGATE`, etc.) |
| `ScriptParseError` | Error enum: `UnexpectedEndOfScript`, `InvalidPushDataLength`, `InvalidOpcode(u8)`, `Other(String)` |
| `opcodes` | Module of Bitcoin opcode constants: `OP_0`/`OP_FALSE`, `OP_PUSHDATA1/2/4`, `OP_1NEGATE`, `OP_1`-`OP_16`, `OP_DROP`, `OP_2DROP`, `OP_CHECKSIG` |

### PushDrop (BRC-48) (`pushdrop.rs`)

| Export | Description |
|--------|-------------|
| `decode(script: &[u8])` | Decode a PushDrop script into `PushDropDecoded`. Auto-detects lock position (before or after fields) by checking for 33-byte pubkey + `OP_CHECKSIG` at start or end |
| `encode(fields, public_key, lock_position)` | Build a PushDrop script from data fields, a 33-byte compressed public key, and a `LockPosition`. Uses `OP_2DROP` for field pairs and `OP_DROP` for the remainder |
| `create_minimally_encoded_chunk(data: &[u8])` | Produce minimally-encoded script bytes for a data push, following Bitcoin consensus rules: empty/`[0]` → `OP_0`, `[1-16]` → `OP_1`-`OP_16`, `[0x81]` → `OP_1NEGATE`, otherwise direct push or `OP_PUSHDATA1/2/4` |
| `PushDropDecoded` | Decoded result: `locking_public_key: Vec<u8>` (33 bytes) + `fields: Vec<Vec<u8>>` (extracted data) |
| `LockPosition` | Enum: `Before` (pubkey + `OP_CHECKSIG` before fields) or `After` (pubkey + `OP_CHECKSIG` after fields) |
| `PushDropError` | Error enum: `ParseError`, `InvalidScriptStructure`, `MissingPublicKey`, `MissingChecksig`, `Other` |

### PushDrop Script Layout

```
LockPosition::Before:
  [pubkey(33)] [OP_CHECKSIG] [field1] [field2] ... [OP_2DROP]* [OP_DROP]?

LockPosition::After:
  [field1] [field2] ... [OP_2DROP]* [OP_DROP]? [pubkey(33)] [OP_CHECKSIG]
```

Fields are dropped from the stack using `OP_2DROP` (drops two at a time) and `OP_DROP` (drops one remaining), leaving only the pubkey for `OP_CHECKSIG` verification.

## Usage

### Encoding a certificate into a PushDrop output script

Used in `src/handlers/certificate_handlers.rs` to create certificate transaction outputs:

```rust
use crate::script::pushdrop::{encode, LockPosition};

let cert_json = serde_json::to_vec(&certificate)?;
let script = encode(
    &[cert_json],
    &public_key_bytes,  // 33-byte compressed pubkey
    LockPosition::Before,
)?;
// script is the locking script for the transaction output
```

### Decoding a PushDrop script from an on-chain output

Used in `src/identity_resolver.rs` to extract certificate data from transaction outputs:

```rust
use crate::script::pushdrop;

let decoded = pushdrop::decode(output_script)?;
let cert_json = &decoded.fields[0];  // Certificate JSON
let pubkey = &decoded.locking_public_key;  // 33-byte key
```

## Testing

- **Unit tests** in `parser.rs` and `pushdrop.rs`: cover direct pushes, `OP_PUSHDATA1/2`, minimal encoding, round-trip encode/decode for both lock positions and multiple fields
- **Cross-validation tests** in `pushdrop_tests.rs`: decode scripts generated by the TypeScript SDK (`#[ignore]` — requires `pushdrop_test_vectors.json` from `test_pushdrop_cross_validation.ps1`). Also includes Bitcoin standard opcode and encoding boundary tests
